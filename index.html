Headspace

Way back in undergrad, I had the pleasure of working with Curtis Litchfield on an semester-long independent study project. We developed Headspace, a Python library that makes it easy to do real-time positional audio in 3 dimensions. It's written in Python and uses an amazing DSP library called PYO written by Olivier Bélanger.

Based on the principle of Head-Related Transfer Functions (HRTFs), Headspace provides a simple, fast API for developers to add positional audio to their applications. Here is a sample application: 

          
  from pyo import *
  from headspace import *

  # Start the audio server
  s = Server().boot()
  s.start()

  # Take a PYO WAV object, put it in 3D space, and start playing it
  shaverWav = SfPlayer("shaver.wav", loop=True)
  shaver = HeadSpace(shaverWav, 0, 0).out()
  shaver.ctrl(title="Shaver")

  # Put another sound somewhere else in the space
  backgroundWav = SfPlayer("VictorOrchestra-BlackWhiteRagtime.wav", loop=True)
  background = HeadSpace(backgroundWav, 100, 30).out()

  doorOpenWav = SfPlayer("dooropen.wav", loop=True)
  doorOpen = HeadSpace(doorOpenWav, -20, -10).out()

  phoneRingWav = SfPlayer("phonering.wav", loop=True)
  phoneRing = HeadSpace(phoneRingWav, -150, -20).out()
  phoneRing.ctrl(title="Phone Ring")

  # Create a simple user interface
  s.gui(locals())

Recently, I've received some requests from folks who have used this library in their own research, and I've been trying to make it a little more performant and accurate. One issue that's been causing me some trouble however, is when the setAzimuth and setElevation methods are called rapid-fire, there is a perceptible "pop" when swapping out the impulse responses... My inkling is that moving some of the methods to PYO's TableMorph data structure would eliminate this issue, but doing signal processing is very taxing on my brain, no matter how simple PYO makes it!

In the mean time, check out the source on GitHub today, and submit a pull request if you understand DSP better than I do!
